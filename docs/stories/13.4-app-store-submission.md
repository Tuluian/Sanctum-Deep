# Story 13.4: App Store Submission Preparation

## Status: Draft

## Story

**As a** developer submitting to the App Store,
**I want** all submission requirements met,
**so that** the app passes review and is available for download.

## Acceptance Criteria

1. App icons in all required sizes (1024x1024 + device sizes)
2. Launch screen/storyboard configured
3. App Store screenshots for all device sizes
4. App Store listing content complete
5. Privacy policy URL accessible
6. Age rating questionnaire completed
7. Build uploads successfully to App Store Connect
8. TestFlight beta testing completed
9. App passes App Store Review

## Dev Notes

### App Icon Requirements
Xcode Asset Catalog handles sizes automatically from 1024x1024 source:
- 20pt, 29pt, 40pt, 60pt, 76pt, 83.5pt (at 1x, 2x, 3x)
- App Store: 1024x1024 (no alpha channel)

### Screenshot Sizes (Portrait)
| Device | Size | Required |
|--------|------|----------|
| iPhone 6.7" | 1290 x 2796 | Yes |
| iPhone 6.5" | 1284 x 2778 | Yes |
| iPhone 5.5" | 1242 x 2208 | Yes |
| iPad Pro 12.9" | 2048 x 2732 | Yes |

Landscape: swap width/height if game is landscape-only.

### Keywords (100 chars)

### Common Review Rejection Reasons
1. **Guideline 4.3** - Spam (ensure app has enough content)
2. **Guideline 2.1** - App doesn't function as expected
3. **Guideline 3.1.1** - Payment issues (must use Apple IAP)
4. **Guideline 5.1** - Privacy concerns
5. **Guideline 4.2** - Minimum functionality (games usually fine)

### Checklist Before Submission
- [ ] All crashes fixed
- [ ] IAP products approved in App Store Connect
- [ ] Privacy policy accessible
- [ ] All placeholder content replaced
- [ ] No debug code in production build
- [ ] Analytics configured (optional)
- [ ] Push notifications configured (if used)
